# Практическая работа 7: Контрольные вопросы

**1. В чём разница между редукцией и сканированием?**
Редукция (Reduction) — это агрегирующая операция, которая вычисляет одно итоговое значение из всего массива (например, общую сумму или максимум).
Сканирование (Prefix Sum) — это операция преобразования, которая генерирует новый массив той же длины, где каждый элемент является суммой (или результатом другой ассоциативной операции) всех предшествующих элементов входного массива.

**2. Какие типы памяти CUDA используются для оптимизации редукции и сканирования?**
Ключевую роль играет **Разделяемая память (Shared Memory)**.
Она используется для кэширования блоков данных на уровне мультипроцессора (SM), позволяя потокам внутри одного блока обмениваться промежуточными результатами с минимальной задержкой. Это позволяет выполнять основную часть арифметических операций без обращения к медленной глобальной памяти.

**3. Как можно оптимизировать префиксную сумму на GPU?**

* **Использование Shared Memory**: Чтобы избежать латентности глобальной памяти на каждом шаге.
* **Устранение конфликтов банков (Bank Conflicts)**: Оптимизация шаблонов доступа к разделяемой памяти (например, сдвиги индексов) для обеспечения параллельного доступа.
* **Алгоритмическая эффективность**: Использование алгоритма Блеллока (Blelloch), который выполняет $O(N)$ операций, вместо алгоритма Хилиса-Стила (Hillis-Steele) с $O(N \log N)$ операциями (work-efficient vs step-efficient).
* **Иерархическое сканирование**: Разбиение задачи на локальные сканирования блоков и последующее сканирование сумм блоков.

**4. Приведите пример задачи, где применяется сканирование.**
Сканирование широко применяется в параллельных алгоритмах сортировки и фильтрации данных, например:

* **Stream Compaction (Уплотнение потока)**: Задача фильтрации массива, где нужно оставить только элементы, удовлетворяющие условию, и записать их подряд. Сканирование используется для вычисления новых индексов этих элементов.
* **Radix Sort (Поразрядная сортировка)**: Для определения позиций элементов в "корзинах".
